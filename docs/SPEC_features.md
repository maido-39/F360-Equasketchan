# 수식 기반 곡선(Equation-Driven Curve) 기능 명세서

**대상 환경:** Fusion 360 add-in (목표 사양 — 현행 플랫폼 제약은 본 문서 범위에서 제외)
**문서 성격:** 기능 요구사항 명세 (Functional / Feature Specification)
**버전:** Draft 0.1

---

## 1. 개요

### 1.1 목적
스케치 환경에서 **임의의 수학적 표현식**(삼각·쌍곡·지수·로그·매개변수식·다변수식)으로 정의된 곡선(폐곡선 포함)을, **사용자 파라미터(예: D3)와 완전히 연동**되고 **언제든 재편집 가능한** 형태로 삽입하는 기능을 정의한다.

### 1.2 설계 목표(이상적 타깃)
본 명세는 Inventor `Equation Curve`, NX `Law Curve`, SolidWorks `Equation Driven Curve`, Grasshopper의 장점을 흡수하되, 그들의 한계(쌍곡함수 부재, 단위 균형 강제, 직접 파라미터 참조 제약)를 제거한 **상위호환 사양**을 목표로 한다.

### 1.3 핵심 원칙
1. **Parametric-native** — 곡선은 정적 형상이 아니라, 정의 데이터(식·범위·좌표계)를 보유한 살아있는 객체다.
2. **Fully associative** — 참조하는 모든 파라미터/입력이 바뀌면 곡선이 자동으로 재계산된다.
3. **Round-trip editable** — 생성에 쓴 모든 입력을 손실 없이 재호출·수정할 수 있다.
4. **Unit-aware, not unit-burdened** — 단위 일관성은 시스템이 처리하고, 사용자가 `1 rad *` 같은 보정을 직접 넣지 않는다.

### 1.4 우선순위 표기 (MoSCoW)
- **[M]** Must — 1차 릴리스 필수
- **[S]** Should — 강력 권장
- **[C]** Could — 여력 시
- **[W]** Won't(now) — 차기 범위

## 2. 용어 정의

| 용어 | 정의 |
|---|---|
| 독립변수(parameter variable) | 곡선을 따라 움직이는 매개변수. 기본 기호 `t` |
| 도메인(domain) | 독립변수의 평가 구간 `[t_min, t_max]` |
| 설계 파라미터(design parameter) | 모델의 User/Model Parameter(예: D3). 식 안에서 상수처럼 참조 |
| 평가(evaluate) | 도메인을 샘플링해 각 점의 좌표를 계산하는 과정 |
| 피팅(fitting) | 계산된 점군을 스플라인으로 근사하는 과정 |
| 연동(associativity) | 참조 입력 변경 시 곡선이 재평가·재피팅되는 성질 |

## 3. 사용자 및 유스케이스

### 3.1 대상 사용자
기계/제품 설계자, 산업디자이너, 연구자 — 렌즈 곡률, 스프링, 캠 프로파일, 임펠러, 로고/패턴, 음향혼(horn), 수학적 형상 연구 등.

### 3.2 대표 유스케이스
- **UC-1** 사인파를 D3(진폭)·D4(주기)에 묶어 그리고, 이후 D3만 바꿔 진폭을 조정한다.
- **UC-2** 극좌표 `r(θ)`로 장미곡선·심장형 등 **폐곡선**을 그린다.
- **UC-3** 3D 원통/구면 좌표로 원뿔나선·구면나선을 그린다.
- **UC-4** 현수선(catenary, cosh)·로그나선 등 **쌍곡/지수 곡선**을 직접 입력한다.
- **UC-5** 생성된 곡선 끝점에 치수·구속을 걸어 다른 형상과 연결한다.
- **UC-6** 타임라인에서 더블클릭해 식을 수정하고 즉시 반영한다.

## 4. 기능 요구사항 — 입력 모델

### 4.1 입력 모드
- **FR-1.1 [M]** **Explicit 모드**: 한 종속변수를 독립변수의 함수로 정의. 2D 데카르트 `y = f(x)`, 2D 극 `r = f(a)`.
- **FR-1.2 [M]** **Parametric 모드**: 좌표 각 성분을 독립변수 `t`의 함수로 정의. 2D `(x(t), y(t))` 또는 `(r(t), θ(t))`, 3D `(x(t),y(t),z(t))` 등.
- **FR-1.3 [S]** **성분별 혼합 정의**(NX 방식): 3D에서 X·Y·Z 각각에 상수/Explicit/Parametric 법칙을 독립적으로 지정.
- **FR-1.4 [C]** **Implicit 모드**: `f(x,y)=0` 형태를 등위선 추출로 곡선화.
- **FR-1.5 [W]** **다변수 곡면** `z=f(x,y)` (차기 범위 — 본 곡선 기능과 분리된 모듈).

### 4.2 좌표계
- **FR-2.1 [M]** 2D: **데카르트(x,y)**, **극좌표(r,θ)**.
- **FR-2.2 [M]** 3D: **데카르트(x,y,z)**, **원통(r,θ,z)**, **구면(r,φ,θ)**.
- **FR-2.3 [M]** 각도 입력 단위 토글: **radian / degree** (기본 radian). 시스템이 내부 변환을 책임진다.

### 4.3 독립변수와 도메인
- **FR-3.1 [M]** 기본 독립변수 기호 `t`. Explicit에서는 `x` 또는 `a`(극)도 허용.
- **FR-3.2 [S]** 독립변수 기호 사용자 변경 가능(이름 충돌 회피).
- **FR-3.3 [M]** 도메인 `t_min`, `t_max` 입력. **두 경계 모두 설계 파라미터/식으로 표현 가능**(예: `t_max = 2*pi*N`).
- **FR-3.4 [M]** 샘플 수 또는 샘플 밀도(points-per-unit) 지정. 미지정 시 곡률 적응 샘플링(아래 8.2).
- **FR-3.5 [S]** 도메인 방향(증가/감소) 및 폐곡선 여부 플래그.

## 5. 기능 요구사항 — 수식 / 함수 라이브러리

### 5.1 연산자
- **FR-4.1 [M]** 사칙연산 `+ - * /`, 거듭제곱 `^` 및 `pow(a,b)`, 괄호, 단항 부호.
- **FR-4.2 [S]** 모듈로 `mod(a,b)`, 정수화 `floor/ceil/round`, 절대값 `abs`, 부호 `sign`.
  - *주: 이들은 불연속을 유발하므로, 피팅 전 세그먼트 분할 로직과 함께 제공(8.4).*

### 5.2 함수 (★ = 상용 대비 차별화 포인트)
- **FR-5.1 [M]** 삼각: `sin cos tan`, `cot sec csc`.
- **FR-5.2 [M]** 역삼각: `asin acos atan`, **`atan2(y,x)` ★**(상한 구분에 필수).
- **FR-5.3 [M] ★** 쌍곡: `sinh cosh tanh` — **네이티브 지원**(SolidWorks엔 없어 e^x 재작성 필요).
- **FR-5.4 [S]** 역쌍곡: `asinh acosh atanh`.
- **FR-5.5 [M]** 지수/로그: `exp` (e^x), `ln`, `log`(상용로그), `log(base, x)`, `sqrt`, `cbrt`.
- **FR-5.6 [S]** 보조: `min max hypot`, 선형보간 `lerp`.
- **FR-5.7 [C]** 특수: `gamma`, 베셀 `besselJ` 등(연구용).

### 5.3 상수
- **FR-6.1 [M]** `pi`, `e` 내장.
- **FR-6.2 [S]** 황금비 `phi` 등 추가 상수.

### 5.4 표현식 규칙
- **FR-7.1 [M]** 대소문자 구분 여부를 **명확히 정의**(권장: 함수는 소문자, 파라미터는 원형 유지).
- **FR-7.2 [M]** 화이트리스트 기반 안전 평가(임의 코드 실행 차단).
- **FR-7.3 [S]** 식 입력 중 실시간 문법 검증 및 오류 위치 하이라이트.
- **FR-7.4 [C]** 긴 식 편집용 확장 입력창(외부 에디터 붙여넣기 호환).

## 6. 기능 요구사항 — 설계 파라미터 연동 (★ 최우선 요건)

- **FR-8.1 [M]** 식 안에서 **User/Model Parameter를 이름으로 직접 참조**(예: `D3 * sin(t)`). SolidWorks식 우회 없이 직접 허용.
- **FR-8.2 [M]** 참조된 파라미터에 **의존(dependency) 등록** → 해당 파라미터 변경 시 곡선이 **자동 재평가**된다.
- **FR-8.3 [M]** `t_min/t_max`, 샘플 수, 변환(7장) 필드도 파라미터/식 참조 가능.
- **FR-8.4 [S]** 곡선 정의에 사용된 **로컬 상수를 파라미터로 승격**하여 Change Parameters에 노출(이름·단위·설명 지정).
- **FR-8.5 [S]** 단위 차원 자동 검사: 좌표 출력은 길이 차원, 각도 입력은 각도 차원으로 일관성 검증. **불일치 시 사용자에게 보정 제안**(자동 `1 rad`/`1 mm` 삽입을 시스템이 수행).
- **FR-8.6 [M]** 순환 참조 검출 및 거부.

## 9. 기능 요구사항 — 재편집 / 연동 (★ 최우선 요건)

- **FR-11.1 [M]** 곡선은 **타임라인 노드**로 남는다.
- **FR-11.2 [M]** 노드 **더블클릭/우클릭 Edit** 시, 생성에 쓴 **모든 입력이 그대로 복원된 편집 대화상자**가 열린다(식·범위·좌표계·단위·변환·샘플링).

## 11. 오류 처리 / 검증

- **FR-13.1 [M]** **정의역 특이점** 검출: `1/0`, `ln(≤0)`, `tan`/`sec`의 점근선, `asin`의 정의역 밖 등. 해당 점을 건너뛰거나 구간을 분할.
- **FR-13.2 [M]** `t_min ≥ t_max`, 빈 도메인, NaN/Inf 산출 시 명확한 오류.
- **FR-13.3 [S]** 미지원 함수/미정의 식별자 사용 시 구체적 안내.
- **FR-13.4 [S]** 과도한 샘플 수에 대한 성능 가드(상한·경고).
- **FR-13.5 [M]** 평가 실패 시에도 기존 형상을 파괴하지 않고 롤백.

## 12. 비기능 요구사항

- **NFR-1 정밀도** — 기본 피팅 편차 목표 ≤ 모델 단위의 1e-3(공차 조정 가능).
- **NFR-2 성능** — 기본 해상도에서 곡선 1개 평가·피팅 < 200 ms 목표. 프리뷰는 인터랙티브(끊김 없는 갱신).
- **NFR-3 견고성** — 특이점·고곡률·폐곡선·대규모 샘플에서 크래시 없음(Equation Driven Surface 류의 고해상도 크래시 회피).
- **NFR-4 결정성** — 동일 입력은 항상 동일 형상 산출(무작위 샘플링 금지, 적응 샘플링은 결정적 규칙으로).
- **NFR-5 단위 안전** — 모든 입력은 모델 단위계와 일관되게 해석·변환.
- **NFR-6 국제화** — 소수점·인자 구분자 로캘 처리, 메시지 다국어 대응.

## 13. 수용 기준 (Acceptance Criteria)

다음 시나리오가 모두 통과해야 한다.

1. **사인파 연동** — `y = D3 * sin(D4 * x)` 입력 후, D3를 두 배로 바꾸면 진폭이 두 배가 된다(재실행 없이).
2. **극좌표 폐곡선** — `r = 1 + cos(a)`(심장형)을 닫힌 단일 곡선으로 생성, 자기교차/끊김 없음.
3. **3D 나선** — 원통좌표 `r=D1, θ=t, z=D2*t` 로 등간격 나선 생성, D2 변경 시 피치 변화.
4. **쌍곡 곡선** — `y = D3 * cosh(x / D3)`(현수선)을 e^x 재작성 없이 직접 입력·생성.
5. **재편집 무손실** — 1번 곡선을 더블클릭하면 원래 식·범위가 그대로 복원되고, 식만 바꿔 즉시 반영.
6. **특이점 안전** — `y = tan(x)` 를 `[-π, π]`로 줘도 점근선에서 크래시 없이 분할 처리.
7. **구속 호환** — 생성 곡선의 끝점에 coincident/치수 부여 후 모델이 정상 재계산.

## 14. 프리셋 곡선 카탈로그(참고 — FR-12.4 구현 데이터)

| 이름 | 모드/좌표 | 식 |
|---|---|---|
| 사인파 | Explicit 데카르트 | `y = A*sin(k*x)` |
| 아르키메데스 나선 | Parametric 극 | `r = a + b*t, θ = t` |
| 로그(등각) 나선 | Parametric 데카르트 | `x = a*exp(b*t)*cos(t), y = a*exp(b*t)*sin(t)` |
| 장미곡선 | Explicit 극 | `r = A*cos(n*a)` |
| 심장형(cardioid) | Explicit 극 | `r = A*(1 + cos(a))` |
| 현수선(catenary) | Explicit 데카르트 | `y = c*cosh(x/c)` |
| 인벌류트 | Parametric 데카르트 | `x = R*(cos t + t*sin t), y = R*(sin t − t*cos t)` |
| 사이클로이드 | Parametric 데카르트 | `x = R*(t − sin t), y = R*(1 − cos t)` |
| 리사주 | Parametric 데카르트 | `x = A*sin(a*t + δ), y = B*sin(b*t)` |
| 원뿔나선 | Parametric 원통 | `r = c*t, θ = t, z = h*t` |
| 구면나선 | Parametric 구면 | `r = R, φ = t, θ = k*t` |

## 15. 차기 범위 (Out of Scope, 차후)

- 다변수 곡면 `z=f(x,y)` 직접 솔리드/서피스화.
- 데이터 테이블(CSV) 기반 곡선.
- 식의 시간 애니메이션/스윕.
- 곡선군(family) 일괄 생성(파라미터 스윕).
