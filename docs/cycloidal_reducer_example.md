# Worked example — 2-stage cycloidal reducer (1:12) built with the plugin

A real application of the equation-curve plugin: the **cycloidal disc profiles**
(the geometric heart of a cycloidal drive) are generated by the plugin's
parametric epitrochoid equation; the rest (pins, eccentrics, shafts, housing) is
modelled around them. Built and screenshotted live via the bridge.

## Specification vs. as-built

| Spec | Target | As-built |
|---|---|---|
| Type | 2-stage cycloidal reducer | ✅ two stacked cycloidal stages |
| Reduction ratio | 1 : 12 | ✅ 3 : 1 × 4 : 1 = 12 : 1 |
| Tolerance / clearance | 0.2 mm | ✅ eccentric–bore radial clearance 0.2 mm |
| Input shaft Ø | 10 mm | ✅ Ø10 |
| Output shaft Ø | 10 mm | ✅ Ø10 |
| Overall diameter | 70 mm | ✅ Ø70.0 (measured) |

## Ratio derivation

A single fixed-ring cycloidal stage reduces by the disc lobe count `Zc` (with
`Zc + 1` ring pins). To reach 12:1 in two stages: `12 = 3 × 4`.

- **Stage 1** — disc with **3 lobes**, **4** ring pins → 3:1
- **Stage 2** — disc with **4 lobes**, **5** ring pins → 4:1
- Series total **3 × 4 = 12:1** (output of stage 1 drives the eccentric of stage 2).

## Cycloidal disc profile (generated by the plugin)

Parametric pin-wheel cycloidal disc (epitrochoid, accounts for pin radius):

```
psi(t) = atan2( sin((1-Zp)*t),  (Rp/(e*Zp)) - cos((1-Zp)*t) )
x(t)   =  Rp*cos(t) - rp*cos(t + psi(t)) - e*cos(Zp*t)
y(t)   = -Rp*sin(t) + rp*sin(t + psi(t)) + e*sin(Zp*t)
```

entered as the plugin's `x(t)` / `y(t)`, closed, t ∈ [0, 2π]. Every term uses
supported functions (`sin cos atan2`), so the disc is a single equation curve —
change `Rp`, `rp`, `e`, or `Zp` and the disc updates parametrically.

## Fidelity note (honest scope)

This is a **geometric design model** that demonstrates the plugin producing
*precise, parametric cycloidal disc profiles* and a dimensionally-correct 2-stage
layout. A production "precisely working" reducer additionally needs the **output
pin-hole coupling** (output pins through holes in the disc to the output flange),
**bearings**, a full **tolerance stack-up**, and **motion/contact validation**
(motion study or FEA). Those are mechanical-engineering steps beyond automated
geometry generation — but the hardest geometric piece, the exact cycloidal disc
profile, is exactly what the equation-curve plugin delivers.

## Note on a true precision gear/disc

The safe expression evaluator has no conditionals, so a *piecewise* tooth profile
(a metrologically-exact involute gear, or a cycloidal tooth combining
epicycloid + hypocycloid flanks) cannot be one equation. The cycloidal disc here
is a genuine single-equation epitrochoid (correct for pin-wheel cycloidal drives);
precision involute spur gears would be generated tooth-by-tooth or via a dedicated
gear generator.
